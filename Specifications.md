# Pomodro Timer App

user stories -> requirements --> technical design --> implementation (exact code)

## User stories

1. As an office worker, I want to be reminded to rest when after working for some time, so that I can have less eyes strain.
2. As a student, I want to follow the pomodro techniques to improve the **effectiveness**.

## Requirements

1st story

- functional
  1. The system shall remind users after a period of time working for uninterruptable work.
  2. The system shall remind again after 5 mins if users haven't react to the system.
  3. The system shall remind and interrupt users in a way that he must react to the app for interruptable work.
- non-functional
  1. The interruption should be as smooth as possible.

2nd story

- Functional
  1. The rest session shall comes after the focus session ended. 
  2. The system shall set longer rest session after a quantity of focus session is done.

- Non-functional
